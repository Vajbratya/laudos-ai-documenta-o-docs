# Melhores Práticas e Dicas

Maximize sua produtividade e a qualidade dos laudos com estas dicas essenciais do Laudos.AI, desenvolvidas com base na experiência de milhares de profissionais.

## Escrevendo para a IA

### Descrições Eficazes

<CardGroup cols={2}>
<Card title="Seja Objetivo" icon="target">
Use linguagem clara e direta. Evite ambiguidades que possam confundir a IA.
</Card>

<Card title="Seja Específico" icon="search">
Detalhes precisos geram laudos mais acurados. Quanto mais específico, melhor.
</Card>

<Card title="Use Terminologia Padrão" icon="book">
Termos médicos consagrados são melhor reconhecidos pela IA.
</Card>

<Card title="Estruture a Informação" icon="list">
Organize os achados de forma lógica e sequencial.
</Card>
</CardGroup>

### Exemplos de Boas Descrições

<CodeGroup>
```markdown ❌ Descrição Vaga
Paciente com dor. Foi feito TC. Tem umas coisas no pulmão.
```

```markdown ✅ Descrição Eficaz
TC de tórax sem contraste, indicado por dor torácica. 
Achados: Presença de pequeno pneumotórax à esquerda (aproximadamente 15%), 
sem desvio de mediastino. Pequeno derrame pleural bilateral. 
Pulmões com áreas de escarificação subpleurais em ambos os lados.
```
</CodeGroup>

## Otimizando o Editor

### Atalhos de Teclado

<Steps>
<Step title="Salvar">
`Ctrl + S` (Windows) ou `Cmd + S` (Mac) - Salva o laudo atual
</Step>

<Step title="Copiar">
`Ctrl + C` / `Cmd + C` - Copia o texto selecionado
</Step>

<Step title="Colar">
`Ctrl + V` / `Cmd + V` - Cola o texto copiado
</Step>

<Step title="Desfazer">
`Ctrl + Z` / `Cmd + Z` - Desfaz a última ação
</Step>

<Step title="Refazer">
`Ctrl + Y` / `Cmd + Y` - Refaz a ação desfeita
</Step>
</Steps>

### Configurações de Produtividade

<AccordionGroup>
<Accordion title="Auto-save">
Ative o salvamento automático com intervalo de 1 minuto para nunca perder trabalho
</Accordion>

<Accordion title="Tema Escuro">
Use o tema escuro em ambientes com pouca luz para reduzir cansaço visual
</Accordion>

<Accordion title="Notificações">
Configure apenas alertas importantes para evitar distrações
</Accordion>

<Accordion title="Zoom do Editor">
Ajuste o tamanho da fonte para leitura confortável (110-120% é ideal)
</Accordion>
</AccordionGroup>

## Utilizando a Memória IA

### Construindo uma Boa Memória

<Steps>
<Step title="Seja Consistente">
Use a mesma terminologia para conceitos similares
</Step>

<Step title="Revise Regularmente">
Verifique e edite memórias incorretas ou desatualizadas
</Step>

<Step title="Forneça Feedback">
Avalie as sugestões da IA para melhorar o aprendizado
</Step>

<Step title="Importe Glossários">
Adicione terminologia específica de sua especialidade
</Step>
</Steps>

### Exemplos de Termos para Memorizar

<Info>
**Achados comuns:** "Sem alterações significativas", "Dentro dos limites da normalidade", "Sem evidência de..."

**Técnicas padronizadas:** "Realizada com contraste iodado", "Sem uso de meio de contraste", "Protocolo padrão"

**Conclusões:** "Exame dentro dos padrões de normalidade", "Sugere-se controle em...", "Achados compatíveis com..."
</Info>

## Ditado por Voz

### Técnicas de Ditado Eficaz

<AccordionGroup>
<Accordion title="Ambiente Adequado">
Use em local silencioso, preferencialmente com microfone de qualidade
</Accordion>

<Accordion title="Dicção Clara">
Fale pausadamente e articule bem as palavras médicas
</Accordion>

<Accordion title="Pontuação Verbal">
Diga "ponto", "vírgula", "parágrafo" para estruturar o texto
</Accordion>

<Accordion title="Revisão Imediata">
Sempre revise o texto transcrito antes de prosseguir
</Accordion>
</AccordionGroup>

### Frases Úteis para Ditado

```markdown
"Início do laudo. Ponto."
"Técnica: TC de tórax sem contraste. Ponto."
"Achados: Nenhuma alteração significativa identificada. Ponto."
"Impressão: Exame dentro dos padrões de normalidade. Ponto final."
```

## Monitor de Achados Críticos

### Protocolo de Revisão

<Steps>
<Step title="Verificação Diária">
Consulte o monitor pelo menos 3 vezes ao dia
</Step>

<Step title="Priorização Imediata">
Atenda primeiro os achados de crítica imediata
</Step>

<Step title="Documentação Completa">
Registre todas as ações tomadas
</Step>

<Step title="Comunicação Efetiva">
Notifique a equipe médica apropriada prontamente
</Step>
</Steps>

### Verificação de Qualidade

<Warning>
Sempre confirme os achados críticos nas imagens antes de tomar ação. A IA é poderosa, mas o julgamento médico é essencial.
</Warning>

## Segurança e Conformidade

### Boas Práticas de Documentação

<CardGroup cols={2}>
<Card title="Privacidade do Paciente" icon="shield">
Nunca inclua informações que possam identificar pacientes inadequadamente
</Card>

<Card title="Terminologia Padronizada" icon="clipboard">
Use termos médicos reconhecidos e aprovados
</Card>

<Card title="Revisão Médica" icon="user-check">
Sempre revise laudos antes de assinar - IA é ferramenta de apoio
</Card>

<Card title="Documentação Completa" icon="file-text">
Inclua todos os achados relevantes, mesmo os negativos
</Card>
</CardGroup>

### Evitando Erros Comuns

<AccordionGroup>
<Accordion title="Sobreconfiança na IA">
Nunca assine um laudo sem revisão médica completa, por mais bom que pareça
</Accordion>

<Accordion title="Omisão de Achados">
Inclua todos os achados, mesmo que pareçam irrelevantes - podem ser importantes
</Accordion>

<Accordion title="Termos Ambíguos">
Evite termos como "aparentemente normal" - seja específico sobre o que foi avaliado
</Accordion>

<Accordion title="Comparações Ausentes">
Sempre que possível, compare com exames anteriores do paciente
</Accordion>
</AccordionGroup>

## Solução de Problemas

### Problemas Comuns e Soluções

<CodeGroup>
```markdown Problema: IA gerando laudos genéricos
Solução: Melhore suas descrições, use mais memórias, 
revise e edite memórias incorretas
```

```markdown Problema: Ditado não reconhecendo termos médicos
Solução: Fale mais pausadamente, articule melhor, 
use microfone de qualidade, treine o sistema
```

```markdown Problema: Modelos não funcionando bem
Solução: Simplifique a estrutura, use mais campos variáveis, 
teste antes de usar em produção
```

```markdown Problema: Achados críticos não detectados
Solução: Use terminologia padrão, seja específico nos achados, 
revise configurações do monitor
```
</CodeGroup>
